Single Prediction

You will find a form that will allow you to input the necessary data for the model to make a prediction. You will see the result in a flash message and this data will be automatically added to the database.
Some details about the input fields:

  • Full Name: Text
  • Age of the client: Numeric. Must be between 18 and 100.
  • Type of job: Select input.
  • Marital Status: Select input.
  • Education: Select input.
  • Has Credit in Default?: : Select input.
  • Average yearly balance: Numeric, in EUR.
  • Has a housing loan?: Select input.
  • Has a personal loan?: Select input.
  • Contact type: Select input.
  • Last Contact Day: Last day of the month the client was contacted. Numeric. Must be between 1 and 31.
  • Month of the contact: Last month the client was contacted. Select input.
  • Duration of the contact: Numeric, in seconds.
  • Number of contacts: For the current campaign. Numeric.
  • Days After Last Contact: Number of days passed since the contact from a previous campaign. -1 means the client wasn't contacted on a previous campaign. Numeric.
  • Number of Contacts Before: Number of contacts performed before the current campaign. Numeric.
  • Previous Outcome: Outcome from the previous campaign. Select input.

What kind of data was used for the training?

A bank dataset from a previous marketing campaign was used which contains the following:

Client Data:

  • Age of the client
  • Type of job
  • Marital Status
  • Education
  • If they had credit on default
  • Average yearly balance
  • If the had a housing loan
  • If the had a personal loan

Related to last contact:

  • Contact type (cell or telephone)
  • Day of the contact
  • Month of the contact
  • Duration of the contact

Other atributes:

  • Number of contacts for the campaign for each client
  • Number of days passed since the contact from a previous campaign
  • Number of contacts performed before the campaign
  • Outcome from the previous campaign
  • Outcome of this campaign (target variable)

What is the model's performance and how is it measured?

The model selection and hyperparameters were selected using a grid search cross validation.
The model trainig and metrics are obtained using a train/test split are the following:

  • Accuracy score: 79.3%
  • Precision score: 79.3%
  • Recall score: 79.4%
  • F1 score: 79.3%
