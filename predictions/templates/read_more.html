What is CapitalCall?

CapitalCall is an app that uses a machine learning model to analyze client data and make predictions about their likelihood of taking a term deposit. The model takes into account various factors such as demographic information, financial history, and previous banking behavior. By considering these factors, the model can accurately predict the likelihood of a client taking a term deposit when contacted.

Which model is used?

CapitalCall uses a Logistic Regression model, which is a type of statistical model used for binary classification problems, where the goal is to predict a binary outcome (e.g. yes/no, true/false, 1/0).
Logistic Regression models the relationship between the dependent variable and one or more independent variables by using a logistic function, also known as the sigmoid function. The logistic function maps any real-valued number to the range of 0 to 1, which can be interpreted as a probability.
In logistic regression, the parameters of the model are estimated using maximum likelihood estimation, which involves finding the values of the parameters that maximize the likelihood of observing the data given the model. Once the parameters have been estimated, the model can be used to make predictions for new data points.
One of the advantages of logistic regression is that it can handle both continuous and categorical independent variables. Additionally, it is easy to interpret the coefficients of the model, as they represent the change in the log-odds of the dependent variable for a unit change in the independent variable, holding all other variables constant. This makes logistic regression a useful tool for identifying the most important predictors of a binary outcome.

What kind of data was used for the training?

A bank dataset from a previous marketing campaign was used which contains the following:

Client Data:

  • Age of the client
  • Type of job
  • Marital Status
  • Education
  • If they had credit on default
  • Average yearly balance
  • If the had a housing loan
  • If the had a personal loan

Related to last contact:

  • Contact type (cell or telephone)
  • Day of the contact
  • Month of the contact
  • Duration of the contact

Other atributes:

  • Number of contacts for the campaign for each client
  • Number of days passed since the contact from a previous campaign
  • Number of contacts performed before the campaign
  • Outcome from the previous campaign
  • Outcome of this campaign (target variable)

What is the model's performance and how is it measured?

The model selection and hyperparameters were selected using a grid search cross validation.
The model trainig and metrics are obtained using a train/test split are the following:

  • Accuracy score: 79.3%
  • Precision score: 79.3%
  • Recall score: 79.4%
  • F1 score: 79.3%
